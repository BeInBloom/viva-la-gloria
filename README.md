# eoj-card-generator

`eoj-card-generator` - локальный HTTP-сервис на Rust/Axum для сборки PDF-листов из изображений карточек. Проект поднимает простое веб-приложение, отдает каталог карт из `manifest.json` и генерирует готовый PDF для печати по списку `card_id`.

## Что умеет

- показывает каталог карт с курсорной пагинацией;
- отдает preview-изображения для интерфейса;
- собирает PDF из выбранных карт;
- поддерживает ручной ввод ID карт;
- ограничивает частоту запросов на генерацию PDF;
- автоматически очищает старые сгенерированные файлы.

## Стек

- Rust 2024
- Axum + Tower + Tokio
- Moka для rate limiting cache
- `fpdf` для генерации PDF
- HTML/CSS/JavaScript без фронтенд-фреймворка

## Структура проекта

- `src/main.rs` - запуск HTTP-сервера на `127.0.0.1:8080`
- `src/app.rs` - сборка приложения, статика и DI
- `src/http/` - роуты, DTO, middleware и обработчики
- `src/repo/` - чтение карт из `manifest.json`
- `src/service/pdf/` - генерация PDF и очистка `generated-pdf/`
- `static/` - клиентский интерфейс
- `manifest.json` - каталог карт и пути к ассетам
- `assets/` - исходные изображения и preview-файлы
- `generated-pdf/` - папка для готовых PDF

## Требования

- установленный Rust toolchain с поддержкой edition 2024;
- доступный `manifest.json`;
- изображения карт по путям из `manifest.asset_root`;
- preview-изображения по путям из `manifest.preview_root`;
- права на запись в `generated-pdf/`.

Текущий `manifest.json` содержит `311` карточек.

## Быстрый старт

```bash
cargo run
```

После запуска откройте:

```text
http://127.0.0.1:8080/
```

Главная страница позволяет:

- выбрать карты из каталога;
- указать количество копий;
- вставить список ID вручную;
- получить ссылку на готовый PDF.

## HTTP API

### `GET /cards`

Возвращает страницу каталога.

Query-параметры:

- `after` - необязательный курсор; запись с этим ID не включается в следующую страницу;
- `limit` - размер страницы; по умолчанию `24`, максимум `100`.

Пример:

```bash
curl -sS "http://127.0.0.1:8080/cards?limit=3"
```

Пример ответа:

```json
{
  "items": [
    {
      "card_id": "001",
      "title": "flame-magus",
      "preview_url": "/previews/eoj/main_sets/set_1/001__flame-magus__variant-base__rev-02.jpeg"
    }
  ],
  "next_cursor": "001"
}
```

### `POST /pdf`

Генерирует PDF по списку карточек. ID автоматически нормализуются до ширины в 3 символа, поэтому `1` превращается в `001`.

Пример:

```bash
curl -sS \
  -H "content-type: application/json" \
  -d '{"card_ids":["1","12","203"]}' \
  http://127.0.0.1:8080/pdf
```

Пример ответа:

```json
{
  "path": "/generated-pdf/cards-1710000000000.pdf"
}
```

Основные ограничения:

- пустой список `card_ids` отклоняется;
- максимум `100` ID на один PDF;
- таймаут генерации - `2` секунды;
- одновременно выполняется не более `4` задач генерации;
- для `POST /pdf` действует cooldown по IP на `10` секунд после любого запроса, который не завершился клиентской ошибкой; таймаут тоже ставит cooldown.

### Статические маршруты

- `/` - веб-интерфейс
- `/static/*` - frontend-ассеты
- `/previews/*` - preview-изображения
- `/generated-pdf/*` - готовые PDF

## Хранение PDF

Сгенерированные файлы складываются в `generated-pdf/`. Фоновый очиститель:

- удаляет PDF старше `10` минут;
- запускается каждые `10` секунд;
- дополнительно удаляет самые старые файлы, если размер каталога превышает `512 MiB`.
